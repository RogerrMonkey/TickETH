1. Project Overview
1.1 Project Name

TickETH

1.2 Core Idea

TickETH is a blockchain-based NFT ticketing platform that enables secure, transparent, and fraud-resistant event ticket issuance and verification using a Layer-2 blockchain (Polygon recommended for production).

The system ensures:

On-chain ticket ownership

Off-chain scalability

Secure event-day verification

Organizer approval workflows

Admin governance

Regulatory compliance readiness (DPDP Act India)

2. Core Objectives

Eliminate ticket fraud and duplication.

Provide verifiable on-chain ownership.

Enable scalable, low-cost ticket minting.

Provide secure check-in with two-step verification.

Support mass adoption in India.

Be cost-efficient and arm64 compatible.

Be compliant with Indian DPDP regulations.

6. Full System Workflow
6.1 Organizer Workflow
Step 1: Organizer Registration

Fill organizer request form

Stored in DB (organizer_requests)

Status: pending

Step 2: Admin Approval

Admin:

Reviews request

Approves or rejects

DB updated

Organizer role activated

Step 3: Create Event

Organizer:

Inputs title, description, venue, date

Defines ticket tiers

Uploads banner

Chooses network (Polygon)

Backend:

Deploys contract OR uses factory

Saves event in DB

Step 4: Event Goes Live

Status:

Event visible in listings

Tickets available for mint

6.2 Attendee Workflow
Step 1: Browse Events

No login required

Step 2: Connect Wallet

MetaMask / WalletConnect

Must be on Polygon

Step 3: Mint Ticket

User clicks Buy:

Wallet signs transaction

NFT minted

Contract emits event

Backend reconciles

DB ticket record created

Ticket now:

On-chain (ownership)

Off-chain (mirror record)

Step 4: View My Tickets

Displays:

NFT image

Token ID

Status

Event details

Transfer option

Step 5: Transfer (Optional)

User signs transfer transaction

NFT transferred

DB updated

6.3 Event Day – Check-In Workflow (Mobile)
Step 1: QR Generation

Ticket shows:

Dynamic QR

Contains:

ticket_id

nonce

expiry

backend signature

Step 2: Volunteer Scan

Scanner:

Reads QR

Sends data to backend

Backend verifies:

Nonce valid

Ticket exists

Not checked-in

Step 3: Attendee Confirmation

Push notification:
“Confirm check-in?”

User taps Confirm:

Signs short message

Backend verifies signature

Step 4: Entry Granted

Backend:

Marks ticket checked_in

Logs timestamp

Sends confirmation to scanner

Volunteer sees green approval.
